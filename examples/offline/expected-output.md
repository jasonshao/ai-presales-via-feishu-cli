## 1. Customer Background And Needs

### Business Context

Beanlight Tea operates ~120 mall-counter stores and wants to launch a branded WeChat mini-program where members can pre-order, pay, and earn loyalty points. They have an existing POS but no online ordering. They want to know whether we can help them ship a payment-enabled mini-program in 6-8 weeks.

### Core Pain Points

- Branded WeChat mini-program for pre-order pickup
- In-app payment for orders
- Member identity tied to existing phone-number-based loyalty program
- Daily settlement reporting for finance team

### Target Outcome

A WeChat mini-program with payment that the customer can launch under their own corporate WeChat Pay merchant account, with daily reconciliation files matching their finance format.

## 2. Recommended Solution

### Solution Overview

Recommended scenario: **Mini-program payment via API integration** (`PAY_API_MINIPROGRAM`, confidence high). Current report covers 86% of the required confirmation checklist; remaining items are listed in section 4.

### Product Capability Match

- **Mini-program payment SDK + API** — Lets the customer keep full UI control while we handle the payment plumbing: order creation, payment invocation, async result callback, and reconciliation file generation.
  - Constraint: Requires the customer's WeChat Pay merchant account.
  - Constraint: The signing entity for the contract and the merchant account holder must match (subject consistency).
  - Constraint: Refund flow requires merchant-side certificate provisioning.
- **Daily reconciliation and finance export** — Generates daily transaction summaries and detail lines in a customer-defined format so the finance team can reconcile without manual exports.
  - Constraint: Format must be agreed during discovery.
  - Constraint: T+1 delivery is standard; T+0 requires additional architecture review.

### Technical Flow

- Standard order lifecycle: create order -> invoke wx.requestPayment -> server async notify -> reconciliation.
- Daily reconciliation file is generated server-side and pushed to a customer SFTP or downloaded via API.
- Sandbox keys are issued during integration.
- Default outputs: a CSV summary and a CSV detail file per merchant per day.
- Optional: push to customer SFTP, push to a customer S3-compatible bucket, or expose via API.
